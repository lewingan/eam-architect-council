# Evaluation Rubric

Each response is scored 1-5 on five dimensions.

## Dimensions

### 1. Format Compliance (1-5)
- **5:** All required sections present, correct headings, table format for Decision Log.
- **4:** All sections present, minor formatting issues.
- **3:** Missing 1 section or significant formatting deviation.
- **2:** Missing 2+ sections.
- **1:** Output does not follow the template at all.

### 2. Completeness (1-5)
- **5:** Covers architecture components, data model, integration, assumptions, decisions, and next agent.
- **4:** Covers all areas but one is shallow.
- **3:** Missing depth in 2+ areas.
- **2:** Significant gaps in coverage.
- **1:** Barely addresses the question.

### 3. Correctness (1-5)
- **5:** All technical claims are accurate; SAP references are valid; industry practices are sound.
- **4:** Minor inaccuracies that don't affect the recommendation.
- **3:** One notable technical error.
- **2:** Multiple technical errors.
- **1:** Fundamentally incorrect recommendations.

### 4. Assumptions Quality (1-5)
- **5:** Assumptions are explicit, reasonable, and cover key unknowns.
- **4:** Good assumptions, may miss one edge case.
- **3:** Assumptions present but vague or incomplete.
- **2:** Few assumptions listed, important gaps.
- **1:** No assumptions or open questions documented.

### 5. Actionability (1-5)
- **5:** A developer could start building from this output. Clear next steps, concrete entities, defined interfaces.
- **4:** Mostly actionable, may need minor clarification.
- **3:** Provides direction but lacks implementation detail.
- **2:** Too abstract to act on.
- **1:** Not actionable.

## Scoring
- **Total:** Sum of all 5 dimensions (max 25).
- **Pass threshold:** Total >= 15 (average 3/5 per dimension).
- **Excellent:** Total >= 22.
